Система управления логистикой «Хром-КЗ»
Разработайте веб-приложение для департамента логистики компании «Хром-КЗ» на базе Python + Flask. Приложение должно автоматизировать прием и управление заявками на логистику по Астане и Казахстану, а также предоставлять сотрудникам возможность отслеживать свои заказы.
1. Публичная часть сайта (Public UI)
Дизайн должен быть современным, минималистичным, с преобладанием белого цвета и чистых линий. Используйте Bootstrap для адаптивного макета.
•	Главная страница:
o	Простой и понятный интерфейс.
o	Три кнопки: «Отгрузка по Астане», «Отгрузка по Казахстану» и «Отследить заказ».
•	Форма заявки:
o	Открывается по нажатию на кнопки «Отгрузка».
o	Поля: Ф.И.О., номер телефона, email, адреса погрузки и выгрузки с контактными данными, описание и габариты груза.
o	После отправки заявки клиент получает уникальный трекинг-номер, который отображается на экране и отправляется на указанный номер телефона.
•	Отслеживание заказа:
o	На странице отслеживания сотрудник вводит свой трекинг-номер и видит текущий статус заказа.
•	Регистрация:
o	Возможность регистрации для сотрудников. После регистрации и привязки номера телефона сотрудник может видеть историю всех своих заказов в личном кабинете.

2. Административная часть сайта (Admin Panel)
•	Страница входа:
o	Безопасная страница для авторизации сотрудников.
•	Роли пользователей:
o	Сотрудник (Employee): Имеет доступ только к своим заказам. Может редактировать только контактные данные и адреса в своих заявках.
o	Логист (Logist): Полный доступ ко всем функциям. Может просматривать и редактировать все заявки.
•	Страница «Все заявки»:
o	Таблица или список всех заявок с возможностью фильтрации по статусу, дате и типу (Астана/Казахстан).
o	Редактирование заявки: Логист может изменить статус заказа, назначить цену, назначить водителя из выпадающего списка и оставлять внутренние комментарии.
•	Отчетность и аналитика:
o	Отчеты: Количество заявок за период, общая выручка, средняя стоимость доставки.
o	Аналитика: Визуализированные дашборды с графиками: динамика заявок, распределение по статусам, загруженность водителей.
________________________________________
3. Техническая реализация (Tech Stack: Python + Flask)
•	Backend: Python и фреймворк Flask.
•	Database: SQLite с использованием ORM SQLAlchemy.
•	Аутентификация: Flask-Login для управления сессиями.
•	Формы: Flask-WTF для безопасной работы с формами.
•	Telegram-бот:
o	Интеграция с Telegram Bot API через библиотеку requests.
o	При поступлении новой заявки, на канал или в группу логистов автоматически отправляется уведомление с основными данными заказа.
•	Проектная структура:
o	Использовать стандартную структуру Flask-приложения с разделением файлов routes.py, models.py, forms.py и папок templates, static.
•	Интерфейс:
o	HTML-шаблоны на Jinja2.
o	Bootstrap для оформления.
o	Font Awesome для иконок.
1. Отчетность по затратам
Эти отчеты должны предоставлять базовые данные о расходах департамента.
•	Отчет по расходам:
o	Общие затраты: Суммарные расходы на все перевозки за выбранный период (день, неделя, месяц).
o	Средние затраты: Средняя стоимость одной доставки.
o	Разбивка по типам: Затраты на доставки по Астане и отдельно по Казахстану.
•	Отчет по водителям:
o	Затраты на водителя: Общая сумма, выплаченная каждому водителю за период.
o	Эффективность: Соотношение затрат к количеству выполненных доставок каждым водителем. Это поможет выявить наиболее экономически эффективных сотрудников.
________________________________________
2. Инструменты аналитики
Раздел с аналитикой должен быть визуализированным и интерактивным.
•	Дашборд: Стартовая страница, где в виде виджетов и графиков представлены ключевые показатели (KPI).
o	Динамика затрат: График, показывающий изменение расходов за последние месяцы. Это поможет отслеживать тренды.
o	Карта затрат: Интерактивная карта, на которой отображаются точки погрузки и выгрузки. Затраты могут быть визуализированы, например, размером маркера, что покажет, какие направления наиболее дорогие.
•	Отчет по эффективности:
o	Анализ затрат на маршрут: Сравнение стоимости перевозки по разным направлениям. Например, можно сравнить средние затраты на доставку в Караганду и в Шымкент.
o	Анализ затрат на единицу груза: Средние затраты на перевозку 1 кг или 1 кубического метра груза. Это поможет оптимизировать тарифы и выявлять неэффективные перевозки.
